# Development Standards

## 1. Branching

1. main: stable branch
2. feature/*: feature development
3. fix/*: bugfix

## 2. Commit Convention

1. feat: new feature
2. fix: bug fix
3. docs: documentation
4. refactor: refactor
5. test: tests

## 3. API Rules

1. RESTful path naming
2. Unified response format
3. Global exception handling

## 4. Quality Gates

1. Lint must pass
2. Unit tests must pass
3. PR review required
